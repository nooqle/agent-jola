## Design Context

### Users
AgentPoppy is for developers and AI-Agent power users who want to configure a local Agent, give it a character and battle prompt, then watch it compete in a four-player shrinking-zone arena.

### Brand Personality
Playful, tactical, and developer-native. The product should feel like a polished indie game launcher with credible developer tooling underneath, not like a generic SaaS dashboard.

### Aesthetic Direction
Use a refined pixel-game lobby direction: dark tactical surfaces, warm gold command accents, authentic Social Chameleon character art, crisp map/grid details, and restrained motion. Avoid hand-drawn CSS mascots, cheap neon, overbusy dashboards, generic glass cards, and placeholder-looking animations.

### Design Principles
- Real character assets beat synthetic CSS character drawings.
- The first screen should communicate "waiting room + local Agent setup" in one glance.
- Prefer UI-native lobby scenes over poster-like illustrations: map preview, ready slots, and one empty join slot are clearer than decorative battle art.
- Motion should feel cinematic and purposeful: slow drift, scanner sweeps, safe-zone pulses, not frantic bouncing.
- Keep UI copy short and human-readable; explain the local Agent relationship clearly.
- Preserve accessibility: strong contrast, responsive layout, and reduced-motion support.
